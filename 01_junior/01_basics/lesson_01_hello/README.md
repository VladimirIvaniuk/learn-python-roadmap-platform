## Теорія

### 1. Виведення — `print()`

`print()` — вбудована функція Python для виведення тексту у консоль.

```python
print("Hello, World!")        # Hello, World!
print("Привіт, Python 🐍")    # Привіт, Python 🐍
print(42)                     # 42
print(3.14)                   # 3.14
```

**Декілька аргументів** — об'єднуються пробілом:
```python
print("Ім'я:", "Аліса")         # Ім'я: Аліса
print("2 + 3 =", 2 + 3)         # 2 + 3 = 5
print("a", "b", "c")            # a b c
```

**Параметри `print()`:**
```python
# sep — роздільник між аргументами (за замовчуванням " ")
print("2026", "03", "01", sep="-")    # 2026-03-01
print("a", "b", "c", sep=" | ")      # a | b | c

# end — що ставити в кінці рядка (за замовчуванням "\n")
print("Крок 1", end=" → ")
print("Крок 2", end=" → ")
print("Крок 3")
# Крок 1 → Крок 2 → Крок 3

# flush — одразу вивести у консоль (корисно при логуванні)
import time
print("Завантаження...", end="", flush=True)
time.sleep(1)
print(" Готово!")
```

### 2. Змінні

**Змінна** — це іменований контейнер для збереження значення в пам'яті.

```python
name = "Дмитро"     # рядок (str)
age = 25            # ціле число (int)
height = 1.80       # число з крапкою (float)
is_active = True    # логічний тип (bool)

print(name)         # Дмитро
print(age + 5)      # 30
```

`=` — це **присвоєння**, не математична рівність. Він бере значення справа і зберігає у змінну зліва.

**Змінна може змінюватися:**
```python
score = 0
print(score)    # 0
score = 100
print(score)    # 100
score = score + 10
print(score)    # 110
```

**Кілька присвоєнь:**
```python
x = y = z = 0               # усі три = 0
a, b, c = 10, 20, 30        # розпакування
first, *rest = [1, 2, 3, 4] # first=1, rest=[2, 3, 4]

# Обмін значеннями без тимчасової змінної (тільки Python!)
a, b = b, a
```

### 3. Правила іменування — PEP 8

PEP 8 — офіційний стандарт стилю Python. Його дотримуються всі серйозні проєкти.

```python
# ✅ Змінні та функції — snake_case
user_name = "Іван"
total_price = 199.99
is_logged_in = False
max_retry_count = 3

# ✅ Константи — UPPER_SNAKE_CASE
MAX_CONNECTIONS = 100
DATABASE_URL = "sqlite:///app.db"
PI = 3.14159265

# ✅ Класи — PascalCase (вивчимо пізніше)
# class UserProfile:
# class HttpRequestHandler:

# ❌ Неправильно
userName = "Іван"         # camelCase — не Python-стиль
TotalPrice = 199.99       # схоже на клас
t = 199.99                # незрозуміла назва
x1 = "Іван"              # теж незрозуміло
```

**Правила в одному реченні:** змінні — `snake_case`, константи — `UPPER_CASE`, класи — `PascalCase`.

### 4. Коментарі

Коментарі Python ігнорує при виконанні. Вони — для людей (включаючи себе через місяць).

```python
# Це однорядковий коментар
x = 5  # коментар після коду (відступ — 2 пробіли)

"""
Це рядок документації — docstring.
Зазвичай пишуть на початку модуля, функції або класу.
Python зберігає їх у атрибуті __doc__.
"""

'''
Можна також з одинарними лапками,
але подвійні — стандарт (PEP 257).
'''
```

**Золоте правило:** коментуй **ЧОМУ**, а не **ЩО**:
```python
# ❌ Погано — пояснює очевидне
i = i + 1  # збільшуємо i на 1

# ✅ Добре — пояснює причину
i = i + 1  # пропускаємо заголовок (індекс 0) при обробці даних

# ✅ Добре — пояснює неочевидне
timeout = 3.5  # AWS Lambda тримає з'єднання максимум 3.5 секунди
```

### 5. F-рядки — форматування тексту (Python 3.6+)

F-рядки — найчистіший і найшвидший спосіб форматувати текст.

```python
name = "Олена"
age = 29
city = "Харків"
balance = 1234567.89

# Звичайна підстановка
print(f"Привіт, {name}! Тобі {age} років.")
# → Привіт, Олена! Тобі 29 років.

# Вирази всередині {}
print(f"Через 5 років тобі буде {age + 5}")
print(f"Ім'я великими літерами: {name.upper()}")
print(f"Ти з {city}? {'Так' if city == 'Харків' else 'Ні'}")

# Форматування чисел
print(f"Баланс: {balance:,.2f} грн")       # 1,234,567.89 грн
print(f"Pi = {3.14159:.3f}")               # Pi = 3.142
print(f"Прогрес: {0.756:.1%}")             # Прогрес: 75.6%
print(f"Двійкова: {42:b}")                 # 101010
print(f"Ширина поля: {name:>10}")          # "     Олена"

# Налагодження з = (Python 3.8+)
x = 42
print(f"{x=}")          # x=42  — дуже зручно для дебагінгу!
print(f"{x * 2 = }")    # x * 2 = 84
```

**Не використовуй старий стиль:**
```python
# ❌ Старий style — складно читати
"Привіт, %s! Тобі %d років." % (name, age)

# ❌ .format() — довго писати
"Привіт, {}! Тобі {} років.".format(name, age)

# ✅ F-рядки — чисто і швидко
f"Привіт, {name}! Тобі {age} років."
```

### 6. Введення від користувача — `input()`

```python
# input() завжди повертає рядок (str)!
name = input("Як тебе звуть? ")
print(f"Привіт, {name}!")

# Числа треба конвертувати
age_str = input("Скільки тобі років? ")
age = int(age_str)        # str → int
print(f"Через 10 років тобі буде {age + 10} років")

# Коротша форма
price = float(input("Введи ціну: "))    # відразу конвертуємо
```

> ⚠️ Важливо для цього курсу: у веб-ранері платформи `input()` може викликати `EOFError`,
> бо там немає інтерактивного stdin. Це нормально.
>
> Використовуй два режими:
>
> **1) Platform-режим (для кнопки Run/Check):**
> ```python
> # Замість input() — фіксовані тестові дані
> name = "Іван"
> age_str = "25"
> age = int(age_str)
> print(f"Привіт, {name}! Через 10 років тобі буде {age + 10}")
> ```
>
> **2) Local-режим (термінал, реальний ввід):**
> ```python
> name = input("Як тебе звуть? ")
> age_str = input("Скільки тобі років? ")
> age = int(age_str)
> print(f"Привіт, {name}! Через 10 років тобі буде {age + 10}")
> ```

### Типові помилки початківців

```python
# ❌ Помилка 1: NameError — опечатка або змінна не визначена
useer_name = "Іван"
print(user_name)    # NameError: name 'user_name' is not defined

# ❌ Помилка 2: забув лапки
print(Hello)        # NameError: Python думає, що Hello — змінна

# ❌ Помилка 3: TypeError — рядок + число
age = 25
print("Вік: " + age)           # TypeError!
print("Вік: " + str(age))      # ✅ перетвори у рядок
print(f"Вік: {age}")           # ✅ або f-string

# ❌ Помилка 4: input() повертає str, не int
year = input("Рік: ")
print(year + 1)     # TypeError: can only concatenate str (not "int") to str
print(int(year) + 1)  # ✅

# ❌ Помилка 5: одинарне = замість порівняння ==
# if x = 5:   # SyntaxError — у Python це заборонено
if x == 5:     # ✅
    pass
```

### У реальному проєкті

В продакшені `print()` замінюють на **logging**:

```python
import logging

# Налаштування логів (зазвичай у config.py)
logging.basicConfig(level=logging.INFO, format="%(asctime)s %(levelname)s %(message)s")
logger = logging.getLogger(__name__)

# Різні рівні важливості
logger.debug("Деталі для розробника")      # тільки при DEBUG
logger.info(f"Запит від користувача")      # звичайна інформація
logger.warning("Підозріла активність")     # попередження
logger.error("Не вдалось підключитись")    # помилка, але програма продовжує
logger.critical("БД недоступна!")          # критично, треба діяти зараз
```

Константи — для конфігурації застосунку:
```python
# config.py
DATABASE_URL = "sqlite:///app.db"
SECRET_KEY = "super-secret-key-change-in-prod"
MAX_CONNECTIONS = 20
ALLOWED_HOSTS = ["localhost", "myapp.com"]
```
